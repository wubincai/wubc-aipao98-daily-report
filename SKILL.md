# 爱跑98竞赛日报表生成器

## 流程概述

1. 获取日期
2. 检查 / 引导填写配置
3. 确认数据源
4. 运行脚本生成报表
5. 告知结果

---

## 第1步：获取日期

询问用户要生成哪一天的报表。

```
用户说: "帮我生成7月10日的爱跑98竞赛日报"
→ 提取日期: "7月10日"
```

日期格式统一为 `#月##日`（如 `7月10日` 或 `12月25日`）。

---

## 第2步：检查配置文件

配置文件位于: `scripts/config.json`（相对 SKILL.md 所在目录的 scripts/ 子目录下）

### 2.1 如果配置存在

直接读取并使用。提示用户配置内容。

### 2.2 如果配置不存在（首次使用）

引导用户填写以下信息：

**时间段配置** — 请用户按顺序列出所有时间段，格式如 "06:30-08:00"。没有限制数量。

逐段提问，每段问：
- 时间段名称（如 "08:00-14:00"）
- 这个时间段有几个班组？
- 每个班组有几个人？

存储格式示例：
```json
{
  "name": "08:00-14:00",
  "teams": [{"members": 3}, {"members": 3}]
}
```

**基础奖励** — 所有组员统一的基础奖励值（如 0.25）。

**保存配置** — 将填写的内容保存为 `scripts/config.json`，格式：
```json
{
  "config_version": 1,
  "base_reward": 0.25,
  "periods": [
    {"name": "00:00-03:00", "teams": [{"members": 1}]},
    {"name": "3:00-6:00", "teams": [{"members": 1}]},
    {"name": "6:00-6:30", "teams": [{"members": 2}]},
    ...
  ]
}
```

**后续使用**：下次再调用此 skill 时，直接读取已有配置，并询问用户是否要修改。如需修改则重新引导填写。

---

## 第3步：确认数据源

### 3.1 数据源存放位置

默认路径: `~/Documents/加油站数据表/`

如果目录不存在，创建它：
```bash
mkdir -p ~/Documents/加油站数据表
```

### 3.2 查找数据文件

文件名格式: `deal_daily_excel.jasper (##月##日).xls`

全路径: `~/Documents/加油站数据表/deal_daily_excel.jasper (7月10日).xls`

### 3.3 检查并引导

- 如果文件存在 → 继续
- 如果文件不存在 → 提示用户从管控系统下载交易记录明细，按此规则命名并放入该目录：

  > 请登录加油站管理系统，进入"加油交易查询"界面，设定查询日期为指定日期（全天 00:00-23:59），点击导出，选择 Excel 格式。
  > 下载后重命名为 `deal_daily_excel.jasper (##月##日).xls`，放入 `~/Documents/加油站数据表/` 文件夹。

首次使用本 Skill 时提示"数据源位置说明"，后续不再提示。

---

## 第4步：运行生成脚本

### 4.1 执行命令

```bash
SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
python3 "$SKILL_DIR/scripts/generate_report.py" \
  --date "7月10日" \
  --config "$SKILL_DIR/scripts/config.json" \
  --data-dir ~/Documents/加油站数据表 \
  --output-dir ~/Documents/爱跑98竞赛日报表
```

其中 `--date` 的值替换为第1步提取的日期。

### 4.2 输出

生成的文件在 `~/Documents/爱跑98竞赛日报表/` 目录下，文件名格式为：
`爱跑98竞赛日班组对抗赛统计表（##月##日）.xlsx`

---

## 第5步：告知用户

通知用户：
- 报表已生成，查看位置
- 打开后需在 F 列（组员）和 H 列（系数）填写数据
- I 列（基础奖励）已预填，可按需修改
- 右表会自动根据左表 F 列数据计算
